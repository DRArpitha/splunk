应用 -- 您现在使用的就是一个应用

Splunk 是由各种不同的应用所组成。Splunk 的各种应用都是通过视图，仪表板和配置文件等组件，让您可以从不同的角度来分析及显示您的 IT 数据。 您现在正在使用的就是一个应用 - Splunk 的 "Getting Started" 应用。Splunk 提供了很多应用供您选用，例如：Splunk 就特别为您当前正在运行的操作系统设计了一个专用应用。您在部署 Splunk 时包含了 Splunk 自制或内建的应用。 默认情况下，Splunk 就内建了 "Search" 应用，在入门介绍中所提供的所有范例，就是用 "Search" 应用作为范例, 将通过弹出窗口来显示（如果您使用的是 Firefox 浏览器，则会显示在新的标签页中）

数据索引

只要是机器产生的数据，Splunk 就有办法检索这些 IT 数据。没错，就是这样 -- 任何数据，无论是来自哪里或不同的格式，Splunk 就可以为这些数据建立索引，不需要通过任何其它定制的解析程序、通讯连接程序或适配器。您可以提供任何数据给 Splunk，包括 Unix服务器和网络设备的系统日志、微软 Windows 系统的事件日志、定制应用程序的日志、甚至是配置文件以及系统指标等， 也就是说，整个运行状况清晰地呈现在您的眼前。

上传本地文件

如果您想上传一个本地计算机上的文件给 Splunk 系统，可通过 Splunk 的管理页面来进行。

  1. 点击[管理] » 数据导入 » 文件与目录
  2. 点击[新增]
    3. 进入新增页面...
  3. 选择"上传本地文件"
  4. 请根据该页面提示，设置其他选项

实时监控文件或目录异动

设置 Splunk 读取哪些正被其他程序或设备写入数据的文件。这可以让您实时监控应用程序的日志，包括 J2EE 或 .Net 应用程序、网站访问日志等。Splunk 会持续在数据写入这些文件或目录时建立索引，而这些文件或目录必须能被当前运行的 Splunk 服务器能够访问得到。

  1. 点击[管理] » 数据导入 » 文件与目录
  2. 点击[新增]
    3. 进入新增页面...
  3. 选择"监控文件或目录异动"
  4. 请输入需要监控文件或目录的完整路径，例如，在微软的Windows系统下，您可以输入：c:\apache\apache.error.log 来监控本地文件，或 \\hostname\apache\apache.error.log 来监控远程主机文件。在 Unix 系统下，输入 /var/log 来监控本地文件，或 /mnt/www01/var/log 来监控远程主机上的目录
  5. 请根据该页面提示，设置其他选项

通过网络通信端口接收数据

使用这个方式可以接收通过 TCP 或 UDP 通讯端口发送的原始数据，例如：设置 Splunk 通过监听 UDP 514 通讯端口来接收系统日志（syslog）的数据。

  1. 在 Splunk [管理]页面上点击"数据导入"链接
    2. 进入数据导入页面...
  2. 点击 UDP 这一行的"新增"链接，添加 UDP 通讯端口来接收数据
  3. 点击 TCP 这一行的"新增"链接，添加 TCP 通讯端口来接收数据li>
  4. 设置 Splunk 将会监听的通讯端口
  5. 请根据该页面提示，设置其他选项

专用应用的数据导入

默认情况下，根据您的操作系统，Splunk 事先内建了一个与操作系统相对应的专用应用，如：微软 Windows 系统或 *nix （Unix, Linux）。 这些专用应用可以让您监控系统性能、事件日志、以及文件系统和注册表的异动。您可以同过启用这些专用应用来导入数据。您也可以从 SplunkBase 下载其他更多的应用来满足您的数据导入，例如：Blue Coat。

切换并启用 "Windows" 或 "*NIX" 应用：

  1. 点击 Splunk 网页右上角的[应用]
  2. 从下拉列表中点击"Home"选项，然后选择[Splunk主页面]
  3. 选择 "Windows" 或 "*NIX" 应用（如果您使用的是微软 Windows 系统，那么您将只能看到 "Windows" 应用，如果您使用的是 Unix 平台，那么您将只能看到 "*NIX" 应用）
  4. 如果应用没有被启用，请点击[启用]来启用该应用
  5. 请按照上述步骤进行配置，来满足您的系统需求

其他导入方式

除了上述方式外，还有其他方式把数据导入 Splunk 。以下是一些常用的方式：

脚本导入

创建一个${_("scripted input")}来处理您的特定数据。脚本导人对于收集在命令行下运行命令产生的数据是非常有用的，例如：收集 vmstat、iostat、netstat、top 等命令所产生的数据。通过脚本导人可以借助 APIs 、其他远程数据接口、消息队列来收集数据，也可以收集系统产生的系统指标和象 vmstat，iostat 等系统命令产生的状态数据。在 SplunkBase 上有许多应用就是用脚本导人来收集特定应用系统的数据，通过Splunk 管理页面，你就可进行脚本导人的配置。

监控文件系统的变化

想了解您的文件系统有什么变化吗？您可以启用${_("file system change monitoring")}，只要发生变化就可以立即知道。使用这种方就可以监控系统的重要文件和配置文件的变动，符合法规遵从的要求，同时发现是否存在系统层面的影响及未经授权的问题来满足系统安全及维护上的考量。

分布式架构的数据收集

您了解如何从一个分布式架构系统中将数据（例如：一群应用程序或网站服务器的本地日志）导人 Splunk 吗？我们可以将 Splunk 配置成一个轻量级的转发器，实时地收集数十台或数百台甚至上千台服务器的数据，然后将数据发送到一台主要的 Splunk 服务器进行检索。通过 ${_("Splunk forwarders")} 可以将其他系统的数据发送到 Splunk 。请在 Splunk 管理页面配置转发功能

  4. 在输入条件中使用通配符。如果您想寻找登录尝试失败的事件，在事件中可能会包含 "failed" 或 "failure" 的信息，那么您可以输入 "fail* login" 进行搜索：

交互式筛选结果

您可以象使用时间轴那样进行交互式操作与浏览搜索结果。在本节中，您将体验到只需点击一下鼠标，就可以添加或删除搜索条件进行搜索。

  1. 进入 "Search" 应用。
    2. 进入搜索页面...
  2. 输入一个任意条件进行搜索
  3. 将您的鼠标移至搜索结果显示区，请注意当鼠标滑过那些字或词组时就会突出显示，就表示您可以将这些字或词组添加到您的搜索条件。
  4. 在您的搜索结果中单击一个突出显示的字或词组，这个字或词组就会添加到您原来的搜索条件并显示在搜索栏中，然后，在以前的搜索结果中过滤掉不符合附加条件的数据。
  5. 当您再用鼠标单击一下搜索结果中已经是突出显示的字或词组时，突出显示就会消失，同时，Splunk 就会将这个字或词组从您的搜索条件中移除并更新搜索结果。
  6. 此外，您可以指定排除条件，将符合排除条件的数据剔除。将鼠标移至那个突出显示的字或词组，然后按住Alt键(如果是Windows系统，则按住Ctrl键)单击以下鼠标，Splunk 就会将这个字或词组连同布尔运算符 "NOT" 直接添加到您的搜索条件中并更新搜索结果。

字段搜索

虽然任意搜索非常简单并且功能强大，但通常所获得的结果可能不是您想要的。例如，您想要除 HTTP 200 以外的事件，如果您只输入 "NOT 200" 进行搜索，您可能同时也将您想要的事件剔除了。例如一个来自 IP 地址中包含了 "200" 的 "HTTP 503" 的事件就会因为不符合搜索条件而被剔除。

当 Splunk 在为原始数据中每一项数据建立索引时，Splunk 会根据字段名称与对应值得组合、标题或其他意义明显的数据来新增字段。例如，Splunk 会自动增加数据来源主机、来源和来源类型等字段。Splunk 也能识别数据中的其他信息，如 IP 地址、HTTP 状态码等。用户也可以自定义字段，您可以在"Splunk 知识"单元中找到这方面的信息。

  1. 进入 "Search" 应用。
    2. 进入搜索页面...
  2. 例如，搜索网站活动的相关数据：
  3. 注意在搜索结果左侧的字段菜单。这些字段都是 Splunk 自动解析产生的。

    4. 在搜索结果区所显示的字段都列在 "已选字段(Selected fields)" 标题下。您可以自行增加想要显示的字段。其他 Splunk 自动解析出来但没有显示在搜索结果的字段，则被列在 "其他相关字段" 的标题下。

  4. 字段菜单里的每一个字段名称旁边都显示一个数字，表示在搜索结果中该字段存在不同的值数量。您可以点击任一字段名称来查看该字段值的排名统计，然后再点击排名统计中的任一字段值，就会将该值添加到您的搜索条件中并更新搜索结果。
  5. 如果您正在搜索的是网站活动相关数据，您可以点击 "挑选字段(Pick fields)"，从弹出的窗口中选择 "status" 字段，请将此字段添加到字段菜单中。
  6. 请注意，"status" 字段所表示的是 HTTP 状态码：200、503、404等。现在，您可以利用这个新增的字段来搜索或排除某些特定值得数据。例如，您想查找所有访问网站失败的事件，您可以输入 "http NOT status=200" 进行搜索：
  7. 您还可以使用比较运算符( >, <, >=, <=)；例如，想要查找状态码大于 300 的所有事件，您可以输入 "status > 300" 进行搜索：

时间轴(Timeline)操作

时间轴是对每一个时间点所发生的事件数的可视化表示。因此，您可以使用时间轴来突现事件的发生或分析事件活动的高峰与低谷。

  1. 进入 "Search" 应用。
    2. 进入搜索页面...
  2. 尝试搜索 "error"，然后注意搜索栏下方的时间轴。
  3. 当时间轴根据搜索结果的变化而更新时，您可能会注意到事件的发生。在时间轴上所显示的高或低分别表示系统活动的高峰期或服务器的停用。
  4. 请在时间轴上点击某一点，然后拖曳鼠标到另一个点。您的搜索结果显示区将随您所选的这个时间范围而更新，只显示此时间范围内所发生的事件。
  5. 请点击时间轴上的某一个柱状体，此时，搜索结果显示区将只显示您点击的时间点所发生的事件。

搜索助手

搜索助手是一个内建的快速参考手册，供用户在建立搜索语句时使用。它提供了搜索命令的详细说明，包括搜索命令的使用范例，以及建议其他可使用的命令。

  1. 进入 "Search" 应用。
    2. 进入搜索页面...
  2. 您可以点击搜索栏下的绿色箭头来打开搜索助手
  3. 如果搜索栏中还没有输入任何命令，搜索助手会简要地介绍 Splunk 搜索及说明如何建立一个搜索语句。默认情况下，搜索助手会根据输入的搜索命令来提供参考信息。
  4. 搜索助手的左侧显示的是搜索命令的简要说明及使用范例
  5. 搜索助手的右侧显示的是您曾使用过的搜索命令，以及列出当前搜索栏中的搜索命令后最常用的命令

Splunk 知识

Splunk 在搜索一个从未处理过的 IT 数据时，能够自动地从数据中提取出有意义的信息并实时地增加您自己的信息。增加关于事件、字段、事务及统计的信息，同时，也可以识别、命名或标注这些信息。

Splunk 在执行搜索时，才会对应检索这些信息，因此，不管何时只要在您需要的时候都可以新增字段或事件类型，并且不需要因此原因而重新建立索引。一起来看看如何搜索某个特定用户的所有相关事件，以及实时地取得某一特定作业的统计数据。

事件类型

当您搜索数据的时候，您通常希望过滤掉您不想要的信息，也就是说搜索出来的资料通常都具有某些共同特性，您可以给这些搜索出来的资料一个共同的名称，或者定义一个“事件类型”。您所定义的“事件类型”会作为值存储在eventtype这个字段里。这表示日后您可以像搜索其他资料一样，直接用eventtype这个字段将这些资料再搜索出来。下面的列子讲演示如何将一个搜索结果定义成一种“事件类型”,然后利用它来搜索出资料。

字段定义

Splunk在建立资料索引时，会自动的分析提取知识，您也可以在任何需要的时候自行新增知识。而且新增知识后并不需要重新建立索引。本节将演示如何使用互动式字段定义功能来提取及保存新字段。

  1. 进入 "Search" 应用。
    2. 进入搜索页面...
  2. 请以主机，数据源，来源类别字段进行一个搜索。任何事件资料的字段定义都需要指定在哪一个主机或数据源或来源类别产生作用。
  3. 从您的搜索结果选择一笔时间资料。
  4. 点击该笔资料时间字段下方的小箭头，打开一个菜单。
  5. 选择“提取字段”选项，开启互动式字段定义窗口。
  6. 根据页面说明输入资料。例如，您可以新增用户姓名, 来源IP地址和目的IP地址等字段。

字段标签

标签可以帮助您分组有相同字段值的搜索结果。标签是为某一字段（如：事件类型，主机，数据源，来源类别）的特殊值定义的名字。例如，可以用服务商名字来标记主机或者标记该主机所遵行的法规规范，如PCI。

总的来说，您可以将标签用作：

  • 帮助您跟踪抽象字段值，如IP地址和ID编号，您可以使用标签来分类标示其所代表的地点和名称。

使用报表生成器

启动报表生成器来产生和格式化您的报表。

  1. 进入 "Search" 应用。
    2. 进入搜索页面...
  2. 请任意执行一个你想要用来产生报表的搜索。
  3. 在时间轴上方的状态栏，有一个“生成报表”的链接，点击链接则可以在一个新的窗口启动报表生成器。(您不需要等到搜索完成后才去启动报表生成器)。
  4. 报表生成器打开“报表内容定义”画面。请使用这个页面所提供的下拉选项设置您的报表参数。 首先是选择报表类别,请注意您所选择的报表类别将会影响您后续报表内容设置的其他选项。
  5. 点击下一步: 当您完成报表内容定义后可以点击此按钮来格式化报表。 Splunk会将会带您进入“报表格式化”画面, 此时所显示的是根据您报表内容定义画面所选择的产生的初稿。
  6. 请点击“报表格式化“来修改报表格式。 例如，假设Splunk所给的是直条图, 儿您觉得改成区域堆叠图看起来比较好, 就可以在这里调整.这里提供了很多格式选项,如，有创意的图表标题，改变图例说明位置，以及其他更多选项。
  7. 这样你就完成了报表的编辑！现在您可以保存报表，或者导出到一个文件，或者打印，或者给一个URL将这个报表分享给其他公司同事。

建立仪表板

通过仪表板的建立，将搜索及报表嵌入网页让用户可以操作使用,这让你生动而有用的报表可以让其他用户查阅。如何做呢？您唯一要做就是事先准备好您想显示在仪表板上的搜索命令。

  1. 进入 "Search" 应用。
    2. 进入搜索页面...
  2. 选择[动作]下拉菜单,点击”新增仪表板“。
  3. 此时，出现"新增仪表板"窗口。请给仪表板一个短名作为仪表板的识别代码，以及一个长名作为显示名称。输入后，请按[建立]按钮。
  4. 一开始，您看到的是一个空的仪表板。请点击”编辑“来开启编辑窗口, 为您的第一个面板选择样式和搜索命令,然后点击[新增面板]将其加入仪表板。
  5. 您可以选择”编辑面板“来更改面板的标题，样式，或者它的搜索内容。
  6. 请重复前面两个步骤在您的仪表板上增加更多面板。你可以拖拽这些面板到您想要的位置。
  7. 这样您就完成了一个仪表板! 请关闭编辑窗口,看看仪表板的效果。如果您觉得有些地方需要修改,可以在本页的上方选择[动作]，点击”编辑仪表板...“来进行更改调整。
